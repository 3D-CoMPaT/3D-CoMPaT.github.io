üìù Annotations#

We provide here some technical details about the format of the dataset and its annotations.

Shapes and styles#

The 3DCoMPaT++ dataset provides 10.009 shapes belonging to 42 distinct categories. We collect mesh annotations for each part instance of each shape, following a consistent nomenclature hand-defined for every shape category in the dataset.

Part annotations are carefully verified through a comprehensive quality checking process, and complemented with material annotations from a list of 13 coarse material categories. Material annotations are later used to sample from a list of 293 fine-grained materials belonging to the coarse categories, in order to generate stylized augmentations of our 3D dataset.

In order: render, part segmentation mask, materials mask, depth map.

Each stylized shape in the dataset is indexed by a code of the following format:

{hex_class_id}_{hex_model_id}__{style_id}

Where:

  • hex_class_id is a 2-digit hexadecimal code for the model‚Äôs shape category

  • hex_model_id is a 3-digit hexadecimal code for the model‚Äôs number within the category

  • style_id is a decimal number indicating the number of the material style augmentation (which we refer to asa composition), ranging from 0 to 1000.

Part annotations#

Comparing coarse-grained (left) with fine-grained (right) annotations.

Part annotations are provided for two semantic-levels: coarse-grained and fine-grained. Fine-grained part classes are defined from a hand-defined shape category-specific nomenclature, while coarse-grained part classes are defined as shape category-specific groupings of fine-grained parts.

For example, in the airplane shape category, the wheel, wheel_connector and wheel_cover fine-grained parts are all merged into the wheel coarse part:

    "airplane": {
        "...": "...",
        "wheel": "wheel",
        "wheel_connector": "wheel",
        "wheel_cover": "wheel"
    },

Shape categories in our dataset can share part names by default. Parts that are category specific and relevant to the category are prefixed by the name of the category (for example: airplane_wing).
